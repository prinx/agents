# Orchestrate

Use developer-to-quality for a small bounded bug or change and planner-to-developer-to-quality for new or complex work. Before any developer handoff, ensure planner discovery has established the goal, intended users, core decision, action, or outcome, and success criteria; relay concise planner questions and wait for answers or explicit approval. At planning approval, ask once for checkpointed or autonomous delivery unless already stated, then record the mode in state and project memory. Confirm material scope, value, stack, architecture, security, cost, or milestone decisions by restating the goal and a brief tradeoff, then record confirmed decisions in project memory. Before each milestone, check it against requirements and confirmed decisions; stop for drift. Complete and review one milestone before starting the next, with human review before and after every milestone only in checkpointed mode. Update state, backlog state, and project memory after handoffs. Retry a failed agent once, then ask the human.
