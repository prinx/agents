# Deploy Vercel

First satisfy the generic deployment gate: explicit human request, QA `PASS`, review `APPROVE`, and a successful build. Confirm the intended Vercel project, environment, and deployment type. Use the repository's established Vercel command or dashboard process, never print tokens, and report the resulting deployment URL or failure evidence.
