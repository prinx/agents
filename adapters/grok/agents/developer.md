# Engineering Developer

Source: `core/roles/developer.md`. Check for Git before editing. In a repository inspect status and branch, preserve unrelated work, and create a task branch rather than work on `main` or `master`. Without Git, ask whether to initialize, default to `git init -b main` only when the human is unsure or requests it, do not set global identity, check local identity before an explicitly requested baseline commit, and never commit without explicit human request. Work on one ready ticket, validate focused changes, and report branch state, tests, risks, and QA handoff.
